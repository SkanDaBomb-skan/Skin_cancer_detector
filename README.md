# DermaVision AI

DermaVision AI is a web-based platform designed to assist in the classification of skin lesions using deep learning. It allows users to upload dermoscopy images and receive an instant AI-generated assessment, detailing the potential diagnosis, confidence level, and overall risk. 

This platform uses a VGG16 deep learning model specialized in binary classification to accurately distinguish between malignant and benign skin lesions. The project is built with Python and Flask, utilizing a modern, responsive web interface.

## Key Features

* Automated Image Analysis: Upload a dermoscopy image and get an immediate classification from the VGG16 model.
* Clinical-Style Reporting: Automatically generate and download professional PDF reports containing the diagnosis, AI explanations, and recommended next steps.
* Dashboard and History: Track past analyses and view aggregate statistics to see the proportion of benign versus dangerous cases over time.
* Secure Access: Protected dashboard and analysis routes secured by an authentication system.
* Clean Architecture: The application logic is separated into database, prediction engine, and routing modules, making it easy to maintain and scale.
* Zero-Configuration Setup: Uses SQLite to store prediction history locally, meaning no separate database server is required to run the application.

## Installation and Setup

To run this project locally, you will need Python 3.11 or newer.

1. Clone or download this repository to your local machine.
2. Open a terminal and navigate to the project directory.
3. Create a virtual environment to manage dependencies:
   python -m venv venv
4. Activate the virtual environment:
   Windows: .\venv\Scripts\activate
   macOS/Linux: source venv/bin/activate
5. Install the required packages:
   pip install -r requirements.txt
6. Run the Flask application:
   python app.py
7. Open a web browser and navigate to http://localhost:5000

## Project Structure

* app.py: The main entry point containing the Flask web routes and authentication logic.
* config.py: Centralized configuration including file upload settings, security credentials, and medical metadata.
* database/db.py: Handles the SQLite database initialization and connections.
* engine/predictor.py: Manages image preprocessing, model architecture reconstruction, and inference logic.
* engine/report.py: Contains the logic for constructing the downloadable PDF reports.
* static/css/style.css: The custom design system and styles for the web application.
* templates/: Contains all the HTML templates for the user interface.
